# Using Moodle's Agentic Development Tools

A plain-English guide for developers.

These tools are designed to help developers contribute to Moodle LMS and build open-source plugins — while protecting the platform from commercial misuse. This guide summarises what you can and cannot do.

> **Note:** This document is a simplified guide only. The full terms are defined in the [Moodle Community License v1.3](LICENSE.md). In the event of any conflict, the License takes precedence. If you are unsure whether your use case is permitted, contact Moodle or seek independent legal advice.

---

## ✅ What you CAN do

### Contribute to Moodle
- Use the tools to develop features, fixes, or improvements for Moodle LMS
- Submit contributions to Moodle core

### Build open-source plugins
- Create plugins using the tools
- Release them under an open-source license
- Share them freely with the community

### Learn and experiment
- Use the tools for personal projects, research, education, and prototyping

### Internal use (organisations and universities)
- Use the tools internally to improve your Moodle instance
- Build plugins or extensions for your own use

> ⚠️ **Contribution condition:** If you build Moodle-related functionality (plugins, extensions, or modifications) for internal use, you must contribute it back to Moodle core or release it as open source within 12 months. See [What you MUST do](#what-you-must-do-organisations) below.

### Offer paid support
- Charge for support services for freely available plugins
- Offer consulting or Moodle implementation services

As long as: the plugin itself is free and open, and payment does not restrict or gate any software functionality.

---

## ⚠️ What you MUST do (organisations)

If your organisation uses these tools to build Moodle-related functionality:

- You must open source or contribute back any plugins, extensions, or modifications to Moodle LMS
- You must do this within **12 months** of first productive use, or **before distributing it externally** — whichever comes first

**This applies to:** plugins, extensions, and modifications to Moodle LMS.

**This does NOT apply to:** general internal tooling, infrastructure code, or systems unrelated to Moodle LMS.

---

## 🚫 What you CANNOT do

### Sell or monetise what you build
- Sell plugins built using these tools
- Charge for access to plugin features
- Gate functionality behind paid tiers or paywalls
- License plugin functionality for a fee

> ℹ️ You can charge for support services — but payment must never restrict access to the software itself.

### Use the tools in commercial products or services
- Use the tools to power a paid product or service
- Provide tool outputs to third parties as a material part of a commercial service offering

### Build competing LMS products
- Use these tools to develop, enhance, or support a commercially offered LMS platform that provides equivalent core functionality to Moodle LMS (course management, user management, assessment delivery)

---

## 💼 Need a commercial license?

If you want to:
- Sell plugins built with these tools
- Use the tools in a commercial product or SaaS offering
- Use them as part of a paid consultancy or managed service

You will need a separate commercial license from Moodle.

To discuss commercial licensing options, contact us.

---

## 🧠 Why this model exists

These tools are designed to:
- Help developers contribute to Moodle LMS
- Accelerate open-source development in the ecosystem
- Ensure improvements flow back into the community

And to protect against:
- Commercial exploitation of tools built to serve the community
- Competitors using Moodle-developed tooling to build rival products
- Plugins being locked behind commercial paywalls
